# Crypto Trading Bot Dashboard — Streamlit MVP

Real-time monitoring dashboard for Phase 3 paper trading test.

## Quick Start

```bash
cd /home/brad/.openclaw/workspace/operations/crypto-bot
./run-dashboard.sh
```

Then open: **http://localhost:8501**

## What It Displays

### 📊 Key Metrics
- **Total Trades**: Count of completed trades
- **Win Rate**: Percentage of winning trades
- **Total P&L**: Cumulative profit/loss across all trades
- **Avg P&L/Trade**: Average profit or loss per trade
- **Losses**: Count of losing trades

### 📋 Recent Trades
- Last 20 trades with timestamps, signals, entry/exit prices
- P&L for each trade (color-coded: green ✅ for wins, red ❌ for losses)
- RSI values and sentiment scores at execution

### 🎯 Signal Analysis
- **Signal Type Breakdown**: Pie chart (LONG, SHORT, HOLD)
- **Trades by Pair**: Bar chart comparing BTC-USD vs XRP-USD

### 💬 X Sentiment Tracking
- **Live sentiment scores** for each pair (BTC, XRP)
- **Sentiment history graphs** showing trends over time
- Score range: -1.0 (bearish) to +1.0 (bullish)

### ⏱️ Test Status
- Start/end times, elapsed time
- Expected completion: 2026-03-28 22:12 PT

## Customization

Edit `dashboard.py` to:
- Change refresh interval: Modify `@st.cache_data(ttl=10)` (currently 10 seconds)
- Adjust chart colors: Update color values in plotly chart definitions
- Add new metrics: Insert new `st.metric()` calls
- Change sentiment thresholds: Modify chart parameters

## Known Limitations

- Dashboard starts empty until Phase 3 writes EVENT_LOG.json
- Sentiment data only appears when X API has fetched tweets
- Auto-refresh may lag slightly during heavy computation

## Troubleshooting

**"No events recorded yet"**
→ Phase 3 test is still initializing. Check process: `pgrep -f phase3_v4.py`

**Dashboard won't start**
→ Ensure venv is active: `source venv/bin/activate`
→ Check port 8501 is free: `lsof -i :8501`

**Sentiment data missing**
→ X API may not be fetching yet. Check x_sentiment_fetcher.py logs.
